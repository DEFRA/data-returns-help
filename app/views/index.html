Before you send your landfill data returns file you need to apply data formatting rules under fixed field headings. This will help you meet Environment Agency data standards, reduce ambiguity and improve consistency in your data returns.

You can set up your data management systems to produce an automated export of data in the correct format then you need to save your file as CSV to send it online.

You can use the online service to submit landfill emissions compliance data covering up to 18 months in the past.

View an example data return.

Landfill compliance monitoring: data formatting rules

These rules explain how you must prepare and present compliance monitoring data for landfill operations according to each field heading in your file.

You must use the short field headings in your file. You can send data under the short headings in any order.

Some fields are mandatory for all data returns. Where possible you should give data for other fields as specified in your permit or licence and particular type of return.

Some fields can only accept entries from the Environment Agency controlled lists of standard terms, which you can download to apply in your software.

Field short heading Description and rules Example
EA_ID

Your EA unique identifier:

  • permits - 2 capital letters and 4 numbers 2 capital letters (eg, AB1234ZZ)
  • Waste Management Licences - 5- or 6-figure number (eg, 654321)

You must complete this field in all data returns. Give the core reference without any separators, slashes or variation numbers.

You can send data returns for multiple permits or licence numbers in one CSV file. Make sure your file is no larger than around 2MB.

 AB1234ZZ or 654321
Site_Name

Your reference for the site. This field allows EA to cross check your site name against EA_ID so you must give a site name.

Only use letters and numbers. Remove unnecessary commas before you save as CSV.

NJG Ltd

Leeds Landfill Site
Rtn_Period

Name of date range covering the entire return. Use the formatting rules in the return period controlled list. Give this information where possible.

To give details of date and time for the individual monitoring item use Mon_Date, see details below.

 Qtr 2 2016
Rtn_Type

You must give the type of data return, to which the row of data relates, based on reporting requirements in your permit or licence.

Select an entry from the return type controlled list. You can return data for more than 1 return type in a single CSV file.

Landfill gas borehole
Mon_Point

You must give a monitoring point reference where the sample was taken. Refer to the sampling or emission point described in your permit or licence. For certain return types monitoring point is the same as the site name.

This field should only contain letters, numbers and spaces. Remove any symbols.

 Borehole 1
Mon_Date

Monitoring date/time. This is the date and, where necessary, time (eg, for a spot sample). If you're monitoring for a period of time (eg, a week), give the date at the end of the monitoring period.

You must give a date in this field, but time is optional. If you're giving time then you can use a T or a space between date and time. Format time as hh:mm:ss using the 24-hour clock.

You must use dashes or slashes between numbers. Format the date as YYYY-MM-DD, using dashes only. You can also use DD-MM-YYYY or DD/MM/YYYY.

2016-08-31 15:28:02

2016-08-31T15:28:04

31-08-2016

31/08/2016
Parameter

You must give the chemical substance or parameter you're monitoring. Select an entry from the parameter controlled list.

A CSV file will insert double quotes if the parameter name includes a comma, eg "1,4-Dioxane".

 CH4
Qualifier

You can use this field to give further definition to a chosen parameter to make it more specific. You can only give an entry from the qualifier controlled list.

Use this field where there’s a basic parameter that can have different values (and different limits) depending on the specifics, eg Dry weight or Wet weight.

More than 1 qualifier can be applied to a parameter. Where possible you should give qualifiers here rather than as a suffix in the Parameter field.

 Filtered
Ref_Period

Give the reference period for the sample, as specified in your permit or licence. This field explains how the sample was taken - eg, 24 hour total, half hour average, monthly mean.

You can only give an entry from the reference period controlled list.

You can leave this field blank if there are no reference period requirements stated in your permit or licence.

 Daily mean
Value

You must give the numerical value of your measurement. As well as numbers you can include the symbols < or>. You can also include 0 (zero). Don’t include any other text characters.

Further information (eg BRT, or Waterlogged) should be put in the Comments field.

Non-numerical values should be entered in the Txt_Value.

 0.2
Txt_Value

A field for text-based response - eg, ‘ Yes’, ‘No’, 'No result' and 'See comments'. Only use ‘See comments’ in this field when the result is descriptive text, eg appearance or site weather conditions. Do not use this if simply adding comments about your result.

You can only give an entry from the text value controlled list.

 No result
Unit

You must give the unit or measure used in your data return. Select an entry from the unit or measure controlled list.

 m
Meth_Stand

Method or standard used for monitoring, which is important for some returns, so you should provide an entry where possible.

You can only give an entry from the monitoring standard or method controlled list.

 BS EN 14791
Comments

Use this optional field for free-text comments about the row of data - eg, if you were unable to take a reading.

You can also use this field if you wish to give your own system references to store with the results.

Avoid unnecessary commas and symbols. You can enter a maximum of 255 characters.

 Borehole blocked

Exceptions and special cases

There are some exceptions or special cases where you may not have to fill mandatory fields because of circumstances. The table below describes these exceptions and explains how you must format your data in these circumstances. It also gives example data.

When Ref_Period is a spot sample or instantaneous reading

If the reference period is left blank we will assume it is either an instantaneous reading or a spot sample. If you prefer to enter the data, you can select "instantaneous" or "spot sample” from the controlled list.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Landfill gas borehole Borehole 1 2016-01-01T09:14 Methane 5

When Ref_Period is multiple measurements in a single calculation.

If you are reporting a measurement with a single figure Reference Period calculated across multiple readings, for example,e.g. an average, max daily or min daily, the Date/Time should be given at the end of the multiple readings. In the example below the result would be the "hourly mean" for 3pm-4pm on 1 August.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Landfill gas engine Eng1 2016-08-01T16:00:00 NOX Hourly mean 380 mg/m3

When reporting an observation as present or absent.

If you are reporting on the presence of a parameter, for example, oil and grease, the Value and Unit fields should be left blank, and you should enter "yes" or "no" in the Txt_Value field.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Emissions to surface water SW1 2016-01-01T09:14 Oil and grease Yes

When no measurement result can be given.

If you are reporting on a parameter but have been unable to provide a result, you should leave the Value field blank, enter "no result given" in the Txt_Value field and enter a reason in the Comments field. You will need to provide one line of data for each parameter you have been unable to measure.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Landfill gas borehole Borehole 1 2016-01-01T09:14 Methane No result Borehole is blocked

When you are reporting a descriptive measurement.

Some parameters need to be described as they have no numerical result, for example,e.g. weather conditions. In this case, leave the Value and Units fields blank and enter a Txt_Value. If none of the other options on the Txt_Value list apply, select "See comments" and enter a comment in the Comments field.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Landfill gas borehole site 2016-01-01 Site weather conditions See comments 11°C sunny breezy 1022 mbs
