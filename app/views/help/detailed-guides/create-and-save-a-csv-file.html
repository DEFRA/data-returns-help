How to format Environment Agency data

Find out how to create and save a CSV file so you can reduce errors and submit your Environment Agency data returns.

Save as CSV

CSV is a file format where each field is separated by a comma.

You can generate a CSV as an extract format from most data management systems. Go to Save as - usually under the File menu - to create a CSV version of your file.

Fields and commas

Once you’ve extracted and saved your data as CSV there should be a comma after every field, except in the last field of the row. The absence of a comma signifies the end of the row.

The commas will be automatically generated when you save your data as CSV.

Each row should have the same number of commas as the number of headings you’ve included in the first row of your file, minus 1 at the end of the row.

If you leave a field blank, you will still see a comma in the CSV as it signifies the beginning of the next field.

The Environment Agency service checks your CSV file for format. Your file will fail if the number of commas is wrong.

When CSV uses double quotes

If there are any commas in your data before you save as CSV - eg, in a parameter name - double quotation marks will be inserted around the data when you export to CSV.

Example

, , ,"1,1,1,2-Tetrachloroethane", , ,

When to remove commas

Before you export your data to CSV you can remove non-essential commas, eg from the names you give to monitoring points in the monitoring point (Mon-Point) field.
