Data rules for EPR compliance monitoring

From:
Environment Agency
First published:
13 January 2016
Last updated:
13 January 2016, see all updates
Part of:
How to format Environment Agency data

These rules explain how you must prepare and present EPR compliance monitoring data according to each field heading in your file.

Field Description Mandatory
EA_ID EA Unique Identifier. This field must be completed in all data returns. Give the core permit reference without any separators, slashes or variation numbers.
For EPR permits the format should be 2 letters and four numbers. (example. AB1234)
For Waste Management Licences it should be a 5 or 6 figure number. (example 654321)
Each line of the csv file will repeat the same permit number. A separate csv file should be submitted for each different permit number.		 Yes
Rtn_Type Return Type. This field must be completed. The Return Type must be taken from the return type controlled list. More than one return type can be reported in a single csv file.
In a csv file this field should be delimited with a double quote in addition to the usual comma delimiter. This is because the return type can contain a comma, which will confuse the software. An example is given below. ,”EPR/IED Sewer, ETP or Tankering emissions return”,		 Yes
Mon_Point Monitoring Point Ref. This is a reference to the sample or emission point at which the sample was taken. The name of the monitoring point should be taken from the permit. It should only contain letters and numbers. Any symbols should be removed. (example Boreholes 1) Yes
Mon_Date Monitoring Date/ Time. This is the date and time for a spot sample. If the monitoring is for a period of time (for example a week), provide the date at the end of the monitoring period. It must be supplied on the format YYYY-MM-DD hh.mm.ss, or DD-MM-YYYY hh.mm.ss. This field must be completed, but the seconds are optional. Yes
Parameter The name of the chemical substance or parameter that is being measured. The name must be taken from the parameter controlled list. Yes
CAS The CAS number is an international standard code for chemical substances. When parameter is selected from the list, the corresponding CAS number should be entered in this field. This field can be left blank if there is no CAS number for the parameter. Must be provided where available
Value The numerical value of the measurement. This should be a number. It can include blanks or zeros. It can also include the symbols < or >. No other text characters are allowed. Any other text, for example "BRT", or "waterlogged", should be put in the Comment heading. In a csv file this field should be delimited with a double quote in addition to the usual comma delimiter. This is because many parameters also contain a comma, which will confuse the software. An example is given below. ,"1,4-Dioxane", Yes
Unit The unit in which the measurement was taken. This field must be completed. Emissions readings should be provided in :v/v or mg/mg3 Temperature should be provided in DegC Pressure should be provided in KPa or mBar. All units must be taken from the units and measures controlled list. There are two measures that do not normally have a unit. The field cannot be left blank so for pH please provide the unit as pH, and for meteorological measurements please provide the unit as “meteorological”. Yes
Ref_Period The reference period for the sample, as specified in the Permit. (example: Hourly mean). The options are given in the reference period controlled list Must be provided where available
Mon_Period The monitoring period name is the name of date range covered by the data return (for example Jan, or Jan-Feb, or Q3 ). Further definition of date range terms is given in the date and time rules. This field only applies to monitoring carried out over a period of time, and it can be left blank for spot samples. The year does not need to be provided in this field, because you will have entered it in Mon_Date. Only for returns that cover more than one point in time i.e. are not spot samples
Method A list of Test Methodologies and Standards is given in the monitoring method and standards rules. This information should be included where applicable. Must be provided where available
Smpl_Ref A reference for the sample can be provided here, to help with analysis of the results. Only letter of numbers are allowed. This field is optional and can be left blank. Optional
Smpl_By The name of who of what the sample was taken by. This field should be left blank unless there is a specific requirement to provide these details, e.g. some permits require you to give the make and model of the monitoring equipment. Optional
Site Name The permit number should allow us to identify the site, but this field allows us to cross check that information. The site name should only contain letters and numbers. Optional
Comments This is an optional field, allowing free text comments about the row of data. Optional
