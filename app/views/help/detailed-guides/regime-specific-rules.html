EPR compliance monitoring rules

These rules explain how you must prepare and present EPR compliance monitoring data according to each field heading in your file.

You must use the short field headings in your file. You can send data under the short headings in any order.

Field Description Mandatory
EA_ID

EA unique identifier. You must complete this field in all data returns. Give the core permit reference without any separators, slashes or variation numbers.

  • EPR permits - 2 letters and 4 numbers (eg, AB1234)
  • Waste Management Licences - 5- or 6-figure number (eg, 654321)

You can send data returns for multiple permits or licence numbers in one CSV file. But make sure your file is no larger than around 2MB.

 Yes
Rtn_Type Type of data return. Use the return type controlled list. You can return data for more than 1 return type in a single CSV file. There are also some return type specific rules in the table below. Yes
Mon_Point

Monitoring point reference. Refer to the sampling or emission point described in your permit (eg Boreholes 1).

It should only contain letters and numbers. Remove any symbols.

 Yes
Mon_Date

Monitoring date/time. This is the date and, where necessary, time for a spot sample. If you're monitoring for a period of time (eg a week), give the date at the end of the monitoring period.
You must give a date in this field, but time is optional. If you're giving time then you can use a T or a space between date and time.
You must use dashes or slashes between numbers. Format the date as YYYY-MM-DD but only uses dashes. You can also use DD-MM-YYYY (or DD/MM/YYYY). Format time as hh:mm:ss.

 Yes
Parameter

Chemical substance or parameter you're monitoring. You must take the name from the parameters controlled list.
A CSV file will insert double quotes if the parameter name includes a comma, eg "1,4-Dioxane".

 Yes
CAS International standard giving codes for chemical substances. When you select parameters from the controlled list you'll see a corresponding CAS number (where available) to enter in your data return file. Provide where possible
Value

Numerical value of your measurement. As well as numbers you can include the symbols < or >. You can also include zeros. Don’t include any other text characters.
Further information (eg BRT, or waterlogged) should be put in the Comments field.

 Yes
Unit

Unit of your measurement. You must give emissions readings in :v/v or mg/mg3. Give temperature in DegC. Give pressure in KPa or mBar.
You cannot leave this field blank. Use 'pH' or 'meteorological' where appropriate.

You must select an entry from the units and measures controlled list.

 Yes
Ref_Period Reference period for the sample, as specified in the permit or licence (eg, Hourly mean). You must select an entry from the reference period controlled list if you give a reference period. Provide where possible
Mon_Period

Standard name for the date range covered by the data return (eg Jan, or Jan-Feb, or Q3).

You must select an entry from the monitoring period controlled list to enter data here for monitoring carried out over a period of time. You can leave blank for spot samples. Don't give the year in this field - you should give that in Mon_Date.

 Only for returns that cover more than 1 point in time - ie, not spot samples
Meth_Stand Method or standard used for monitoring. You must use a term from the monitoring standard or method controlled list. Provide where possible
Txt_Value A field for text-based qualifiers (eg, dry weight). You can only use a qualifier from the text value controlled list. Optional
Smpl_Ref Reference for the sample. Not mandatory but helps with analysis of results, especially to group measurements together (eg, water sampling). Only use letters and numbers. Optional
Smpl_By The name of who or what took the sample. Leave this field blank unless you're required to provide these details in your permit or licence. Optional
Site_Name This field allows EA to cross check your site name against EA_ID. Only use letters and numbers. Remove unnecessary commas before you save as CSV. Optional
Comments Use this optional field for free-text comments about the row of data. Avoid unncessary commas and symbols. Optional

Return type rules

You must apply these specific rules about return types to meet Environment Agency standards and send data returns.

Return type names Field heading and rule
EPR/IED Air (point source) emissions return Unit: always use mg/m³ with hourly for reference period. Measured as per M2 (or such other subsequent guidance as may be agreed in writing with the Environment Agency).
EPR/IED Air (ambient) monitoring return Unit: always use fibre/ml with 2 hour reference period or g/hr for particulates.
EPR/IED Leachate level return Unit: always use monthly for operational cells, or quarterly for non-operational. State whether m>AOD or m>cell base as qualifier in text value field.
EPR/IED Leachate quality return
  • Reference period: use MEPP, quarterly or annually.
  • Unit: must be a concentration, physical parameter (pH, electrical conductivity, COD) or depth in m.
EPR/IED Surface water emissions return
  • Unit: use either flow (l/s or m³/day) or concentration mg/l (exceptions: pH and temperature).
  • Reference period: use either instantaneous or spot sample.
EPR/IED Groundwater quality/level return
  • Unit: use either length (depth), concentration, or physical parameter such as pH, total alkalinity or electrical conductivity.
  • Reference period: use MEPP, MEPP - up gradient, MEPP - down or cross gradient, or spot sample.
  • EPR/IED Landfill gas surface emissions
  • EPR/IED Landfill gas perimeter
  • EPR/IED Landfill gas in waste
  • EPR/IED Landfill gas infrastructure monitoring
 Unit: use %v/v (exceptions - pressures mbr/millibar).
EPR/IED Sewer emissions return Reference period: use spot sample.
EPR/IED ETP or tankering emissions return Reference period: use spot sample.
EPR/IED Landfill annual monitoring report Unit: for a parameter of Energy produced the unit should be MWh. For a parameter of Methane the unit should be 5 v/. For a parameter of available capacity – the unit should be m3. For some other parameters (tbc) – the unit should be a flow measurement.
