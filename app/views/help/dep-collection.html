Data Exchange Protocol

This Data Exchange Protocol sets out a file output specification for transferring regulatory data to the Environment Agency. Use of the protocol, which is designed to facilitate the download of data from existing user systems, removes the requirement to submit information via our existing document templates or online forms, which often have to be manually keyed in by regulated parties at point of data entry.

Background

The Data Exchange Protocol (DEP) offers a straightforward and effective way to share information produced by regulated parties. It does this by focusing on transferring the data values themselves from a customer's existing data systems via the export of a standardised text file ('csv' or 'xml') or a very simple column based spreadsheet. This, which is available across a number of different 'Return Types', replaces the need to complete traditional ‘forms’. While many are now provided in electronic format,  still generally revolve around manual completion of something similar to a paper based product. The methodology also includes the ability for customers to ‘push’ data to us using automated file transfers, as opposed to sending individual emails or uploading files manually through our webpage - though these more traditional methods will also remain available. The Data Exchange Protocol can offer the following benefits

  • Delivers significant burden reduction for regulated business
  • Introduces a more flexible approach with both internal and external benefits
  • Delivers significant cost savings
  • Provides improved data quality
  • Has relatively small set up cost and a short payback period

How it Works

The vast majority of regulatory compliance monitoring data that we require regulated parties to submit, once formatting and presentational aspects have been removed, has a similar structure. This type of data is generally submitted against a permit or other ‘Unique EA Reference’ against which we already hold other information in our permitting or registration systems e.g. the name and address of the operator and of the site/location in question, the company number and SIC code. The data also has a monitoring point or sampling location, which could include ‘the gate’ or the ‘site’ as a whole, which likewise, we already hold information about from data gathered through Permitting. The monitoring reading has a date (and time) or period to which it relates. It has a parameter, a value, a unit and some other pieces of information depending on the type of return. 

Many customers have software which store information in the above way, (monitoring point, date, parameter, unit, value) and by outputting it and submitting it to us within the structure set out below, the Environment Agency will be able to 'mail merge' it to any relevant presentational layout and also transform the data into a format which can be uploaded directly to our data systems using their existing upload mechanisms. e.g. transforming the data into pre-existing xml or Excel spreadsheet formats.

The approach allows operators to ‘dump’ data from their information systems for all or part of their regulatory estate across a number (or all) of their ‘Return Types’. Information is supplied in the same simple format, which means operators can choose how they breakdown their data submission to match their own internal structures and data systems, rather than matching ours. This provides much more flexibility and opportunity for cost savings. The removal of the need to maintain and use expensive, macro based Excel spreadsheet tools and the costs associated with consultation on changes to standard forms  will also provide benefits for the Environment Agency and those we regulate.

The Data Exchange Protocol

The data exchange protocol comprises four elements:

  • A generic file output specification defining the data elements to be included
  • A set of data standards or controlled lists defining allowable values for those elements across all regulatory regimes
  • Regime Specific Rules defining how particular information should be mapped to the generic data elements and providing further refinement of over arching data standards for that regime. These are further divided into Return Type Rules.
  • A mechanism for uploading data to the Environment Agency via a file sharing or file transfer protocol
